The Partnership: Why MPCVault?

Sonic SVM selected MPCVault to support secure, high-velocity operations across treasury, token distribution, and ecosystem incentives. Gaming and consumer apps create frequent, program-heavy transactions—so the wallet layer must support Solana transaction formats (Ed25519 signing), maintain reliability under rate limits, and provide safeguards to prevent costly signing mistakes.

MPCVault delivers institutional MPC multisig controls without sacrificing developer ergonomics. Teams can enforce thresholds, roles, spend limits, and allowlists (programs, mints, and destinations), simulate transactions before signing, and automate repeatable workflows through APIs—while keeping private key shares distributed and exportable to avoid platform lock-in.

Technical Integration Highlights

  • Sonic RPC + gRPC Connectivity: Multi-endpoint routing across Sonic mainnet/testnet (including gRPC support) with rate-limit aware retries and automatic failover.
  • Solana Transaction Support (Ed25519): Full support for Solana transaction formats and Ed25519 signing for Sonic SVM applications, including versioned transactions.
  • Versioned Transactions + Lookups: Support for v0 transactions and Address Lookup Tables enabling complex program interactions with lower transaction costs.
  • SPL Assets + Token Accounts: First-class SPL token handling with automated associated token account management for rewards, mints, and in-game assets.
  • Simulation + Instruction Decoding: Pre-signature transaction simulation and human-readable instruction decoding to reduce signing errors and drain risk.
  • Policy-Driven MPC Multisig: Configurable approval policies including role-based access, threshold signatures, spend limits, and program/mint/destination allowlists enforced before signature generation.

Frequently asked questions

Does MPCVault support Sonic SVM as an MPC-based multisig wallet?

Yes. Sonic SVM selected MPCVault as an infrastructure partner so teams can run MPC-based multisig approvals for treasury, payouts, and operational workflows on Sonic SVM.

What do wallets need to support to work well on Sonic SVM?

Wallets need support for Solana transaction formats (Ed25519 signing), reliable RPC connectivity with multi-endpoint failover, SPL assets and token accounts, versioned transactions, and safe transaction simulation/decoding for program interactions.

How does MPC multisig work on Sonic SVM?

MPC multisig splits private key control into distributed key shares and enforces approval policies (thresholds, roles, and limits). Transactions are constructed and simulated, then only signed once policy requirements are met—ensuring no single party can unilaterally move funds.
